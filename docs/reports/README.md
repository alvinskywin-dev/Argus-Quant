# Reports Archive

Historical engineering reports, organised by type. These are point-in-time
records of work delivered; they are not living documentation. For current
process docs see [`docs/engineering/`](../engineering/).

| Folder | Contents |
|--------|----------|
| [`sprint/`](sprint/) | Sprint deliverables and feature/version milestones (auth, paper trading, vault, multi-exchange, funding engine, entry/stop-loss engines, timezone, landing-page iterations, …). |
| [`fixes/`](fixes/) | Bug-fix, debug, UI, and performance-tuning reports. |
| [`audits/`](audits/) | Source audits, QA/security/Docker reviews, duplicate-signal guard, and dead-code removal. |
| [`deployment/`](deployment/) | Deployment guide, phase-0 deploy notes, and public-deploy security notes. |

> Cross-references inside older reports may cite a report by bare filename
> (e.g. `SPRINT1_REPORT.md`) from when all reports lived in the repo root or a
> flat folder. The file now lives under the matching subfolder above.
